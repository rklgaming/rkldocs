Equipment · Forge · Crafting

RKL Suite — Developer Docs

Three connected FiveM resources for RPG-style gear: wearable equipment with stats and durability, a forge for upgrades, enchants and sockets, and an XP-based crafting system. Runs on ESX Legacy, QBCore and Qbox through one auto-detecting bridge.

rklequipment — gear, stats, combat rklforge — upgrade · enchant · socket crafting — recipes & XP

Overview

Each resource is independent but they share data. The forge reads and writes the equipment metadata, so equipment is the core the others build on.

ResourceDoesOwns
rklequipmentEquip gear, apply stats, durability, combat combosplayer_equipment
rklforgeUpgrade / enchant / socket equipped gearedits equipment metadata
craftingCraft items, levels & XPslv_crafting_data
Load order matters

The forge edits equipment metadata directly, so it must start after equipment. Full order is in the next section.

Install & server.cfg

Required on every framework

ox_lib, ox_inventory, and oxmysql. No es_extended is required for QBCore/Qbox — the bridge detects the framework automatically (qbx_core → Qbox, qb-core → QBCore, es_extended → ESX Legacy).

Ensure order

server.cfg
ensure oxmysql
ensure ox_lib
ensure ox_inventory

# your framework (one of)
ensure es_extended      # or qb-core / qbx_core

ensure rklequipment     # core — start first
ensure rklforge         # AFTER equipment
ensure crafting
awaken_coin on QBCore / Qbox

On ESX a custom getAwakenCoin() is used if present. On QBCore/Qbox the currency routes to an ox_inventory item named awaken_coin — either create that item, or set the currency in config to money, bank, or another item.

Resource names

Exports and cross-resource events use fixed names, and a FiveM export only resolves if the folder name matches the caller. The whole suite is branded rkl* — keep these folder names exactly:

Folder nameUsed by
rklequipmentexports['rklequipment'] · rklequipment:* events
rklforgeexports['rklforge'] · rklforge:* events
rklcraftingrklcrafting:* events & callbacks
All references are rkl

Every export, event, callback and cross-resource sync uses the rkl names above, matching the folders. NUI uses GetParentResourceName(), so the UI follows the folder name automatically. The database tables (player_equipment, slv_crafting_data) keep their original names so existing data is untouched.

rklequipment client exports

The core. Manages equipped gear, the stats they grant (defense, magic defense, set bonuses), durability, and the melee combo system. All of its exports run on the client.

Exports

IsCombatMode()Client

Returns whether the local player is mid-attack (an attack/combo animation is playing). Use it to block actions that shouldn't interrupt combat.

ReturnsTypeNotes
isAttackingbooleantrue while an attack is in progress
client
if exports['rklequipment']:IsCombatMode() then
    return -- player is swinging, skip
end
GetDefense()Client

Current total physical defense from all equipped gear (base + upgrades + sockets). Use it to reduce incoming physical damage.

ReturnsTypeNotes
defensenumber0 when nothing is equipped
client
local def   = exports['rklequipment']:GetDefense()
local final = math.max(1, incomingDamage - def)
ApplyMagicDefense()Client

Current total magic defense from equipped gear. Subtract it the same way you would physical defense for spell / elemental damage.

ReturnsTypeNotes
magicDefensenumbertakes no arguments; returns the value
client
local mdef  = exports['rklequipment']:ApplyMagicDefense()
local final = math.max(1, spellDamage - mdef)
GetSetBonuses()Client

Returns the full set-bonus config table (every set and the stats each grants). Handy for showing set info in a UI.

ReturnsTypeNotes

local sets = exports['rklequipment']:GetSetBonuses()
for setId, set in pairs(sets) do
    print(setId, set.label)
end

Finds which set an item belongs to. Returns the set id and set table, or nil if the item isn't part of any set.

Open files — combos & language

Two files ship outside the escrow so server owners can edit them with the resource still encrypted. Both are listed in escrow_ignore.

language.lua

Every player-facing string lives in a Lang table. Translate values freely; keep any %s placeholder — it's filled at runtime (for example the item name in a "broke" message).

language.lua
['armor_broken_desc'] = 'Your %s broke! Its bonus stats are gone.',

The helper is called _L('key') (not _U) so it never clashes with ESX's built-in translator.

combos.lua

Each weapon type's attack animation sequence is data only — change the dictionary, clip, timing or number of stages without touching logic.

combos.lua
Config.Combos.Katana = {
    [0] = {dict = 'kp_wm_gs_slash_01', anim = 'kp_wm_gs_slash_01_clip', wait = 700, next = 1},
    [1] = {dict = 'kp_wm_gs_slash_02', anim = 'kp_wm_gs_slash_02_clip', wait = 700, next = 2},
    -- last stage loops back with next = 0
}

Built-in types: BigSword, Staff, Unarmed, DualDagger, Katana, DualKatana.

Add a brand-new weapon type

The combo dispatch is data-driven: any weapon type that isn't built in is played by the generic runner — no escrowed edits needed.

  1. Give the weapon item w_type = "yourtype" in config.lua.
  2. Define Config.Combos.yourtype in combos.lua.
combos.lua — custom type
Config.Combos.spear = {
    [0] = {dict = 'melee@large_wpn@streamed_core', anim = 'plyr_takedown_front_attacker',
           wait = 900, flag = 1, next = 1, sound = 'hitflesh', damageRange = 2.5},
    [1] = {dict = 'melee@large_wpn@streamed_core', anim = 'plyr_takedown_rear_attacker',
           wait = 1100, flag = 1, next = 0, freeze = true,
           vfx = {dict = 'scr_sum2_hal', part = 'scr_sum2_hal_rider_weak_green',
                  attachType = 'weapon_tip', loop = true,
                  color = {r = 0.2, g = 0.6, b = 1.0}, scale = 0.5, duration = 700,
                  off = {x = 0.0, y = 1.0, z = 0.0}}},
}

Optional per-stage fields

flag, freeze, blendIn, blendOut, wait, next, sound, damageRange, and vfx (a particle table — add weapon-tip effects straight from the open file).

Optional per-type tuning

combos.lua
Config.ComboSettings = {
    spear = { speedFactor = 0.03, speedCap = 0.30, minWait = 300, comboTimer = 5, slot = 'right_hand' },
}
Built-ins are untouched

The six built-in combos keep their own hand-tuned behavior (Staff casts priest/mage effects, DualDagger freezes, BigSword has VFX + attack speed). The generic runner is used for custom types only.

rklforge server exports

Upgrade, enchant and socket equipped gear. It reads and writes equipment metadata, then tells equipment to refresh. All of its exports run on the server.

Exports

Gives a player a forge requirement item (the Config.ForgeReqItem) carrying the metadata you pass. Use it to hand out forge materials from loot, shops or quests.

ParameterTypeNotes

GetPlayerStatTotal(src, statKey)Server

Sums one stat across all equipped gear — base value plus upgrades plus sockets. Broken items (durability ≤ 0) are skipped.

ParameterTypeNotes
srcnumberplayer server id
statKeystringstat name, e.g. attack, luck, defense
ReturnsTypeNotes
totalnumber0 if none / invalid
server
local atk = exports['rklforge']:GetPlayerStatTotal(src, 'attack')

Convenience wrapper for the total luck stat — the same as GetPlayerStatTotal(src, 'luck'). Useful for loot-chance boosts.

ParameterTypeNotes
srcnumberplayer server id
ReturnsTypeNotes

server
local luck   = exports['rklforge']:GetPlayerLuck(src)
local chance = baseChance + (luck * 0.5)

rklcrafting events: rklcrafting:*

An XP-based crafting station. Players craft items from recipes, gaining levels and XP stored in slv_crafting_data. Internal events use the rklcrafting: prefix.

Export & callback

craft_boost_potion(data, slot)Client

An ox_inventory item-use handler for a craft-boost potion. Reads boostChance from the item metadata and applies a temporary boost to crafting success. Wire it as the item's use export in your ox_inventory items file.

ParameterTypeNotes
datatableox_inventory item-use payload
slotnumberinventory slot
ox_inventory/data/items — client.export
['craft_boost'] = {
    label = 'Crafting Boost Potion',
    client = { export = 'rklcrafting.craft_boost_potion' },
}
lib.callback 'rklcrafting:server:getHunterData'Server

Server callback returning the player's crafting level / XP record. Fetch it from the client with lib.callback.await.

client
local data = lib.callback.await('rklcrafting:server:getHunterData', false)
print(data.level, data.xp)

Integration examples

Common ways other scripts plug into the suite.

Loot luck from equipped gear

Boost a drop chance using the player's total luck (server side):

your loot script — server
local function rollDrop(src, baseChance)
    local luck = exports['rklforge']:GetPlayerLuck(src) or 0
    local chance = math.min(100, baseChance + luck * 0.5)
    return math.random(100) <= chance
end

Reduce custom magic damage by gear

your spell script — client
local mdef  = exports['rklequipment']:ApplyMagicDefense() or 0
local final = math.max(1, spellDamage - mdef)

Don't interrupt an active combo

your script — client
if exports['rklequipment']:IsCombatMode() then
    lib.notify({ description = 'Finish your attack first', type = 'error' })
    return
end

Reward forge materials from a quest

your quest script — server
-- on quest complete
exports['rklforge']:GiveForgeReq(src, 'upgrade_stone', 5, 'Upgrade Stone')
exports['rklforge']:GiveForgeCard(src, 'attack_card', 2)

Troubleshooting

An export returns nil

The folder name must match the export call — see Resource names. exports['rklequipment'] only works if the folder is rklequipment.

Forge doesn't apply stats to gear

Check the ensure order — the forge must start after equipment, and both need oxmysql, ox_lib, ox_inventory running first.

Currency isn't deducted on QBCore / Qbox

Create an awaken_coin ox_inventory item, or change the currency in config to money / bank / another item.

A combo plays no animation

The animation dictionary/clip must exist (a streamed .ycd or a base-game dict). A missing dict means the clip won't play even though the combo logic runs.
